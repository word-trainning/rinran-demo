# りんらん案件 契約書ドラフト生成プロンプト

既存 `contract.html` (契約書雛形) と `vendor-checklist.html` (受注者TODO) と既存実装 `/home/haluu/rinran-crm/` を統合・整合させた契約書改訂版を生成するためのプロンプト。
**A案 (haluu運用 SaaS型) を採用: haluu名義の最安国内VPSで rinran-crm (FastAPI+SQLite+ECharts) を運用、店主はブラウザログインで閲覧+入力。バックアップは haluu が日次取得。**

## 主な改訂方針 (このプロンプトを書き換えた理由)

| 変更点 | 理由 |
|---|---|
| CRM 運用先: **haluu名義の最安国内VPS** | 既存 rinran-crm (FastAPI+SQLite+ECharts) 8割完成を最大活用。haluu が運用責任を持つ |
| 「自活」を**業務的自活**に再定義 | 店主は中華料理店主であって技術者ではない。画面で業務完結=自活、git操作習得=自活ではない |
| 店主アクセス: ブラウザログイン | bcrypt認証、ECharts分析画面、直接入力フォームで完結 |
| OCR: tesseract で着地、精度不足なら別エンジンに切替可能 | 過去注文書50-100枚で精度評価してから判断、契約上は「OCRエンジン」と汎化表現 |
| バックアップ: haluu が日次取得・◯世代保管・別ロケーション | 既存サーバ/プラットフォーム依存を脱却、haluu責任で堅牢化 |
| SLA: 平日10-18時ベストエフォート維持 | 24/7保証はしない。不可抗力(VPS障害・自然災害)も明示免責 |
| 契約終了時引渡し: ソース GitHub push + データCSV + INFRA.md + OPERATIONS.md | A案でも他事業者引継ぎ可能性を担保 |
| 個人情報マスク: 「合理的な範囲で」 | OCR精度依存の絶対表現を回避 |
| 国内サーバ縛り条項: **そのまま維持** | haluu名義の国内VPSで運用するため整合 |
| 損害賠償上限: 直近12ヶ月委託料 | 既存第12条維持 + haluu運用責任明示で再確認 |

## CRM の動作モデル
- 店主は専用URL (例: https://crm.rinran.net) にブラウザからアクセス
- 店舗専用アカウントでログイン (bcrypt認証)
- 注文書をアップロード → サーバ側 OCR (tesseract または同等エンジン) で取込
- 「業種・用途・金額」のみ自動抽出 → 確認画面 → 承認で本登録
- OCR失敗時・電話注文時は直接入力フォームで業種・用途・金額を登録
- 分析画面 (ECharts) で月次/支払/RFM/コホート/サンキー/24区マップ/要望タイプ別を閲覧
- 全ての操作はブラウザ完結 (店主側でのコマンド操作・ファイル管理は不要)

## バックアップ
- 乙(haluu)が cron で日次取得 (DB ダンプ + アップロード済み画像メタデータ)
- 保管: 乙VPS内 + 別ロケーション(乙ローカル または別クラウドストレージ)に複製
- 保管期間: 直近◯日分のローテーション保管
- 月次で別ロケーション分の整合性確認

## CRM (顧客管理) の核心設計 — 個人情報非保持原則
- 注文書はサーバ側OCR (tesseract または同等エンジン) で取り込み
- 業種・用途・金額のみを DB (SQLite) に格納する
- 氏名・電話番号・住所・担当者名・会社名は、合理的な範囲で自動マスク処理を施し、原則として DB に格納しない
- 注文書原本データは取込後の所定期間内に破棄する (再OCRが必要なケースに備え一定期間のみ保持)
- OCR失敗時・電話注文時は直接入力フォームで店主が業種・用途・金額のみ登録できる
- 直接入力フォームでも氏名等の個人情報入力欄は画面に表示しない
- 上記マスク処理は本番運用開始前に実データで動作検証を行う
- 万一マスク漏れが判明した場合は速やかに当該データを削除し、店主に書面で報告する
- データは乙(haluu)名義の国内VPS内でのみ取り扱う
- 食べログ等の外部ポータル規約に抵触する顧客名簿化・二次利用は行わない

## A案 (haluu運用 SaaS型) の優位点と注意点

### 優位点
1. **既存 rinran-crm Python版 (8割完成) を最大活用** — ゼロからの書き直しなし
2. **業務的自活が容易** — 店主はブラウザログインで完結、コマンド操作不要
3. **個人情報非保持 + 国内サーバの両立** — 守秘義務第4条をそのまま満たす
4. **bcrypt認証** — SHA-256ゲートより強い認証
5. **バックアップを haluu が責任持って取れる** — 別ロケーション複製で堅牢化
6. **契約終了時の引渡し書面で他事業者引継ぎ可能性も担保** — A案でも縛り付けにならない

### 注意点 (契約書で対処)
1. **haluu が唯一の運用責任者になる** → SLA 平日10-18時ベストエフォート、不可抗力免責で上限化
2. **VPS事業者障害・自然災害リスク** → 業務範囲外・免責に明示
3. **損害賠償上限** → 直近12ヶ月委託料 (既存第12条維持)
4. **VPS費用の長期見通し** → 月額プランで吸収、大幅値上げ時は協議
5. **海外旅行・体調不良時の対応** → 不可抗力扱い、または信頼できる代行先を事前準備
